Multiple Screens

JGuiD can only create 1 window with a single JPanel. However, with a bit of code editing, it is possible to create a program capable of displaying multiple screens.

After a single window has been designed and exported using JGuiD, another complete window is not normally needed. Instead, design the new window in JGuiD but selectively copy the code and insert it into the code for the first window using a text editor, e.g. Notepad++. The view code screen will help to identify each block of code needed.

Below are some common techniques in Java for having multiple screens:
  1. Multiple Tabs
  2. Multiple JPanels on one screen
  3. Multiple JFrames
(Easy) Multiple Tabs
Using a JTabbedPane, a JPanel can be added to each tab and these tabs can be switched between as needed.

JTabbedPane myTabs = new JTabbedPane();
JPanel panelOne = new JPanel();
JPanel panelTwo = new JPanel();

myTabs.addTab("First",panelOne)
myTabs.addTab("Second",panelTwo)


(Advanced) Multiple JPanels
JPanels are all added to the same location but only 1 JPanel is made visible at any one time. As another panel is needed, the visisble panel is set as invisible and the required panel is now set as visible.

JPanel panelOne = new JPanel();
JPanel panelTwo = new JPanel();

panelOne.setVisible(false);
panelTwo.setVisible(true);


(Messy) Multiple JFrames
Create each frame separately in JGUID. As a button is clicked, it will close itself and open up an instance of another window. This is messy as data cannot be shared between the frames and will involve more file handling operations being called as the user moves between windows.

JFrame windowOne = new JFrame();
JFrame windowTwo = new JFrame();

windowOne.setVisible(false);
windowTwo.setVisible(true);
